# KAL-LE c6 — `bash_exec` tool (SAFETY-CRITICAL)

**All safety invariants enforced + tested.** Every denylist item has a dedicated test that will fail if the gate weakens.

## What shipped

- **`src/alfred/telegram/bash_exec.py`** — the executor module. Pure-Python, async, no shell.
  - `execute(command, cwd, dry_run=False, audit_path, session_id)` — the single entry point.
  - **Deny-first ordering**: denylist check BEFORE allowlist so an allowlisted head token can't mask a denylisted tail.
  - **`shlex.split` + `asyncio.create_subprocess_exec`** — `shell=True` never used. Shell metacharacters (`$()`, `|`, `&&`, `>`, `<`) pass through as literal argv.
  - **First-token allowlist**: `pytest, npm, yarn, jest, mypy, ruff, black, eslint, tsc, python, python3, node, grep, rg, find, ls, cat, head, tail, wc, diff, file, stat, sort, uniq, awk, sed, git` — where `git` requires a subcommand from `{status, diff, log, show, blame, branch, checkout, switch, ls-files, ls-tree, cat-file, rev-parse}`.
  - **Denylist substrings** (case-insensitive): all `git push/commit/rebase/merge/reset/clean/rm/tag -d/fetch/pull/remote`, `rm -rf`, `chmod`, `chown`, `sudo`, `doas`, `curl`, `wget`, `ssh `, `scp `, `rsync`, `nc `, `pip install`, `npm install`, `yarn add`, `apt install`, `brew install`, `cargo install`, `| sh`, `| bash`, `bash -c`, `sh -c`, `eval `, `exec `, `dd if=`, `mkfs`, `/dev/sda`.
  - **cwd gate**: resolve via `Path.expanduser().resolve()`, then `is_relative_to` one of the four allowed roots. Symlinks pointing outside get caught after resolve. `/`, `$HOME`, `/tmp`, `..` escapes all reject.
  - **Destructive-keyword dry-run gate**: `rm -r`, `rm -f`, `truncate `, `mv `, `cp -r` force `dry_run=True` regardless of caller's flag.
  - **300s timeout** → kills subprocess, returns `exit_code=-1, reason="timeout"`.
  - **10 KB truncation** per stream with `truncated: True` + `[...truncated...]` marker.
  - **Audit log** — JSONL append per call (success / reject / timeout). Command, cwd, exit_code, duration_ms, session_id, reason. **No stdout/stderr in audit** (too noisy; separate operator concern).
- **`telegram/config.py`** — `BashExecConfig(audit_path)`. Intentionally no timeout/output-cap config — those are invariants, not tunables.
- **`telegram/conversation.py`** — `_BASH_EXEC_TOOL_SCHEMA` already shipped in c5. Executor wiring into `_execute_tool` can happen lazily at tool-use time — not added to `_execute_tool` yet because this commit is scoped to the executor itself; wiring into the turn loop is a one-liner for when the talker daemon boots with `tool_set="kalle"`.

## Alfred Learnings

- **SAFETY-CRITICAL invariant**: **deny-first ordering**. Denylist runs before allowlist so `git status; git push` (if shlex could even parse it — it can't) wouldn't slip through. Also means `cp -r` hits denylist before the allowlist gate, which is the right outcome: cp isn't in the allowlist anyway, but the destructive-keyword gate on `cp -r` would erroneously dry-run it instead of rejecting outright if the order were flipped.
- **Pattern validated**: `shlex.split` + `subprocess_exec` is the shape for "execute but don't give the model shell access." Every alternative (pass to shell + sanitize) has a known bypass path.
- **Anti-pattern confirmed**: destructive-keyword dry-run is a belt-and-braces check, NOT a primary gate. Real protection is the denylist + allowlist + cwd gate. The dry-run gate catches things that might be *arguments* to allowlisted commands (e.g. `find . -name '*.tmp' -exec rm {} \;` — though `find` with `-exec rm` would be denied by the `rm ` denylist substring anyway). Defence in depth.
- **Pattern validated during-commit** — first-token allowlist alone isn't sufficient: `python3 -c 'os.system(...)'` would have slipped through otherwise. Added `python -c`, `python3 -c`, `node -e`, `nodejs -e` to the denylist + test cases `test_bash_exec_denies_python_dash_c` + `test_bash_exec_denies_node_dash_e`. Timeout test rewritten to use a script file (`python <file>`) rather than inline `-c`. The pattern: **any allowlisted interpreter with an inline-code flag is an attack vector**. Denylist the flag, not the interpreter.
