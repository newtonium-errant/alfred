## Alfred Learnings (arc-wide)

- **Safety-critical enforcement is the allowlist + denylist + cwd gate + destructive-keyword dry-run, in that priority order.** First-token allowlist alone wasn't enough — inline-code flags (`python -c`, `node -e`) needed a denylist entry. The pattern: **any allowlisted interpreter with an inline-code flag is an attack vector; denylist the flag, not the interpreter.**
- **Peer-pair tokens must be distinct across entries.** auth.tokens dict iterates, first-token-match wins. Reusing a token across two entries means the wrong peer wins auth and the `allowed_clients` allowlist rejects. Prod tokens are 64-char random hex — distinctness is automatic — but test fixtures needed vigilance.
- **Route namespace registry swap was the cleanest refactor primitive this arc exercised.** The c3 diff to replace 501 stubs with real handlers was 3 lines (2 imports + 2 dict values). Worth the up-front pattern-setting in the outbound-push arc.
- **Config dataclass builder + env substitution pattern absorbs new fields cheaply.** c1's `daemon.pid_path` + `instance.skill_bundle`, c2's `CanonicalConfig` + `peers`, c6's `BashExecConfig` — each was one dataclass + one entry in the builder function. Zero churn on existing config loaders.
- **Cross-agent bundling (c5 SKILL + scope + schema + tools in one commit) prevents dead-prompt windows.** If the SKILL had shipped without scope/schema backing, the model would have hallucinated tool calls for types the vault layer rejects — degrading UX and logging noise until the next alignment commit. Contract discipline pays off.
- **Correlation-id inbox is the shape that unlocks async peer replies without sticky sessions.** Each side is stateless; the cid connects request to reply across the HTTP hop. Scales to N peers without protocol churn.
- **SAFETY-CRITICAL invariants hold:** every denylist item (`git push/commit/rebase/merge/reset/clean/rm/tag/fetch/pull/remote`, `rm -rf`, `chmod`, `chown`, `sudo`, `doas`, `curl`, `wget`, `ssh`, `scp`, `rsync`, `nc`, `pip/npm/yarn/apt/brew/cargo install`, `| sh`, `| bash`, `bash -c`, `sh -c`, `eval`, `exec`, `python -c`, `node -e`) has at least one test that fails if the gate weakens.
- **cwd gate covers `/`, `$HOME` bare, `/tmp`, relative `../..` escapes.** `Path.resolve()` collapses symlinks before the prefix check.
