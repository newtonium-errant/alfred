## Summary

Distiller's deep extraction pass is now clock-aligned to 03:30
America/Halifax daily. Same pattern as janitor (c3): replaces the
`hours_since_deep >= deep_interval_hours` gate with
`compute_next_fire(deep_extraction_schedule, last_deep)`. 03:30 is
staggered ~1h after the janitor deep sweep at 02:30 so the two
LLM-heavy passes don't collide on rate limits.

Light (1h scan-only) passes stay on rolling interval — only the deep
LLM-invoking pass is schedule-aligned.

## What changed

- `src/alfred/distiller/config.py`
  - Imported `ScheduleConfig` from `alfred.common.schedule`
  - Added `ExtractionConfig.deep_extraction_schedule` with defaults
    03:30 / America/Halifax / daily
  - Kept `deep_interval_hours` as vestigial fallback (unused in the
    daemon when schedule is set — preserved for old config.yaml
    compatibility)
  - Registered `deep_extraction_schedule` in `_DATACLASS_MAP` so the
    recursive config builder picks up the nested dataclass
- `src/alfred/distiller/daemon.py`
  - Imported `compute_next_fire`
  - First-boot `last_deep` now seeded to `now` (not `datetime.min`)
    to avoid firing on every startup
  - Gate: `now >= compute_next_fire(deep_schedule, last_deep)`
  - Added `daemon.deep_extraction_next` log event with ISO target
- `config.yaml` + `config.yaml.example` — new block under
  `distiller.extraction`, old `deep_interval_hours` removed
- `tests/test_distiller_schedule.py` — 6 tests covering defaults,
  load_from_unified behavior (including fallback), first-boot seeding,
  and before/at/after the scheduled fire time

## Alfred Learnings

- **Pattern reused cleanly.** c4 was ~15 minutes end-to-end — the
  janitor migration in c3 was the expensive one because of all the
  decisions (vestigial field, dataclass map, seeding semantics). c4
  copied those decisions. c5 (consolidation) will copy them again.
- **Stagger heavy LLM passes.** 02:30 janitor → 03:30 distiller
  deep → 04:00 distiller consolidation Sundays → 05:55 BIT → 06:00
  brief. All Halifax time. All before Andrew's typical working
  window.
