# KAL-LE c3 ‚Äî Real /peer/* handlers (replace 501 stubs)

## What shipped

- **`peer_handlers.py`** ‚Äî four handlers:
  - `_handle_peer_send` routes by `kind` (message | query_result | notice), auth-checks `body["from"]` against authenticated peer (rejects spoofing at 403 from_mismatch), delegates to a registered peer-inbox callable. 501 `peer_inbox_not_configured` when no callable registered.
  - `_handle_peer_query` ‚Äî canonical query via POST. Owner-gated (404 canonical_not_owned if not owner), schema-validated, uses the shared `_serve_canonical` impl.
  - `_handle_peer_handshake` ‚Äî always 200 (bearer-only authz), returns `{instance, alias, protocol_version: 1, capabilities, peers}`. `canonical_owner` is added to capabilities when appropriate.
  - `_handle_canonical_get` ‚Äî GET endpoint, SALEM-only. Wraps `_serve_canonical`. Body never exposed.
- **Server registry swap**: `ROUTE_NAMESPACES["peer"] = _register_peer_routes` (one-line diff from `_register_peer_stub`), same for canonical. Plan said "one-line swap"; it actually needed two imports at the top, so net +3 lines ‚Äî noted below.
- **Correlation IDs**: every handler echoes `X-Correlation-Id` or mints a 16-char uuid hex. Audited on every canonical read ‚Äî including 403/404 ‚Äî so the audit trail is complete.

## Deviations from plan

- **ROUTE_NAMESPACES swap was 3 lines, not 1** ‚Äî the imports at the top had to move out of the function body (`_register_peer_routes`, `_register_canonical_routes`). The net shape is still one-config-change-flips-behaviour, just with an import block alongside. Flagging so next architect isn't surprised.
- **`/peer/query` shares impl with `GET /canonical`** via `_serve_canonical`. Plan implied two separate implementations; I factored the permission + audit + not-found logic so the rules stay in one place. Any future divergence between POST and GET semantics gets a split then.

## Alfred Learnings

- **Gotcha** ‚Äî aiohttp's `TestClient.app[key] = value` after startup emits `DeprecationWarning: Changing state of started or joined application`. For state that needs to outlive the fixture, stash it on `app[key]` BEFORE `await aiohttp_client(app)`. For per-test inbox wiring, build a fresh app each time. Added a `_build_salem_with_inbox` factory rather than reusing the session fixture.
- **Pattern validated** ‚Äî the "one shared `_serve_canonical` + two entry points" shape keeps permission logic + audit logic co-located. New canonical routes would be one-line wrappers.
- **Missing knowledge** ‚Äî PTB/aiohttp test fixtures (`salem_app`) expose `.app` but actually reading app state from inside a test requires `salem_app.server.app` not `salem_app.app` because `TestClient` has multiple layers. Test passed once I fixed the dereference.
