## Cross-agent contract

- **Builder → prompt-tuner**: `KALLE_CREATE_TYPES` in scope.py defines what the SKILL.md's "Making records" section can claim. If scope narrows later, prompt-tuner pass on SKILL.md must run same-cycle.

## Alfred Learnings

- **Pattern validated** — bundling SKILL.md + scope.py + conversation.py in one commit. No dead-prompt window (SKILL telling the model about tools the code hasn't wired).
- **Anti-pattern avoided** — initial instinct was to bump `KNOWN_TYPES` with pattern + principle for simplicity. That would have leaked coding concepts into Salem's vault-ops layer (e.g. distiller would try to extract `pattern` records from Salem's talker sessions). Separate `KNOWN_TYPES_KALLE` keeps the split clean.
- **Missing knowledge** — the `VAULT_TOOLS` global was imported by tests; renaming to `TALKER_VAULT_TOOLS` would have broken them. Added a legacy alias to avoid churn. Next time any module exports a type/tool list, name it `_BY_SET` from day one.
