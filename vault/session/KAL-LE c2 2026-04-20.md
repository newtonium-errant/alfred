# KAL-LE c2 — Canonical permission config + audit module

Second commit of the KAL-LE MVP arc. Pure machinery + config, no handlers yet — those land in c3.

## What shipped

- **`canonical.py`** — pure `apply_field_permissions()` function. Takes a peer name, record type, frontmatter, and per-peer permissions dict; returns `(filtered_frontmatter, granted_fields, denied_fields)`. Handles:
  - Default-deny (missing peer, missing type, empty fields → nothing)
  - Flat field grants (`name`, `email`)
  - Dotted-field access (`preferences.coding` reads nested + writes nested)
  - Bodies never exposed — the function doesn't even accept a body param
- **`canonical_audit.py`** — append-only JSONL writer + reader. `append_audit()` creates the parent dir, never raises (disk errors log-and-continue).
- **Config**: `TransportConfig` grew `canonical: CanonicalConfig` + `peers: dict[str, PeerEntry]`. `CanonicalConfig` nests `peer_permissions: dict[peer, dict[type, PeerFieldRules]]`.
- **Tests**: 18 new covering default-deny, field intersection, dotted access, STAY-C PHI firewall scenario, audit JSONL write/read.

## Deviations from plan

- `apply_field_permissions` accepts both a raw-dict `type_rules` AND a `PeerFieldRules` dataclass. Plan implied dataclass-only; handler tests in c3 are easier to write with dicts, so the function normalises both. Handler contract unchanged.

## Alfred Learnings

- **Pattern validated** — default-deny as the first branch in every lookup. Zero "did I forget a peer entry?" bugs will be possible because every missing path falls through to `{}, [], all_keys_denied`.
- **Gotcha** — the source_keys list is computed from `list(frontmatter.keys())`. If a peer is granted `preferences.coding`, the raw top-level `preferences` key would naively show as "denied" in the audit log. Added `_not_nested_covered()` helper so the denied list only contains fully-denied top-level keys. Cosmetic but load-bearing for audit-log clarity: operators will read these lines and need "denied" to actually mean denied.
- **Test fixture discipline** — reused the `DUMMY_TRANSPORT_TEST_TOKEN` pattern, no `sk-`/`tg-bot-` prefixes. GitGuardian incident 2026-04-20 informs this.
