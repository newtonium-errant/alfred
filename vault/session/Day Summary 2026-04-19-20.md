## Alfred Learnings

**Bundled session notes work.** After correcting the earlier two-commit-pattern drift on 2026-04-18, every commit in this session bundled code + tests + session note cleanly. Net result: git log is legible, each commit is self-contained, no empty-body "session note for X" commits cluttering the history.

**BIT caught a real bug on first live run.** The first `alfred check --full` surfaced the talker env-var-expansion bug + brief weather 404 — both shipped as hotfixes within the same hour. The check paid for itself before it had even run its first scheduled BIT.

**Free-tier API gatekeeping is a common post-ship blocker.** ElevenLabs' "free tier API access but only for your own cloned voices" was not in my priors; surfaced at first `/brief`; fallback path already shipped so the pipeline didn't die — just went to text. Pattern worth remembering for future paid-API integrations: **always build the "paid feature unavailable" fallback before assuming the free tier covers your use case.**

**Scope-SKILL audit rule caught dead code.** The new team-lead rule from 2026-04-18 fired for real during Q3: a "flesh out the body" SKILL instruction became dead the moment Q3 denied body writes on janitor scope. Prompt-tuner caught it during Q2's SKILL pass. Bundling scope changes with their SKILL audit sibling prevents silent divergence.

**Ratified plans save next-session time.** Both Stage 2b and Stage 3.5 were plan-first passes; both are now in memory with all open questions answered. Next session can go straight to implementation without re-litigating. This is the correct pattern for anything bigger than ~3 commits.
