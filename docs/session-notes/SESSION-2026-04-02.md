## Design Decisions

### Email Triage Architecture
- Categorization happens in n8n Code node (instant pattern matching) rather than Alfred webhook response — filing should be immediate, not dependent on async curator processing
- Living document in vault defines rules; n8n Code node mirrors them for instant filing; curator reads them for deeper AI processing. Two independent paths.
- Used HTTP Request nodes for Graph API calls, not Code node `httpRequestWithAuthentication` (not available in sandbox)
- Native Microsoft Outlook node for mark-as-read (Graph API PATCH not supported via HTTP Request with predefined credentials)

### KAIROS Comparison
- Analyzed Anthropic's leaked KAIROS/AutoDream system and compared to Alfred's architecture
- Alfred's typed knowledge graph is fundamentally stronger than KAIROS's flat-file grep approach
- Three features added to close the gaps: proactive recall, drift detection, consolidation

## Aftermath-Lab Learnings

### New Gotchas
- `this.helpers.httpRequestWithAuthentication` is NOT available in n8n Code nodes — must use HTTP Request nodes with predefined credentials instead
- `fetch()` also not available in Code nodes (already documented, but confirmed again)
- n8n Outlook trigger returns `from` as plain string, not `{ emailAddress: { address: "..." } }` object — Code nodes must handle both formats
- Graph API message update (PATCH) returns 405 via n8n HTTP Request with predefined credentials — use native Outlook node for updates
- n8n Code node may reject returns if function declarations cause sandbox issues — keep code flat

### Anti-Patterns Confirmed
- Using `$('Node Name')` deep in the pipeline instead of Restore Context pattern — fragile and breaks when nodes are renamed
- `continueOnFail: true` without any error visibility — filing failures should at least be visible in execution logs

### Patterns Validated
- Restore Context pattern (Code node after HTTP Request) — mandatory, prevents silent data loss
- `_route` field + Switch node pattern — clean conditional routing
- Code-node-first pattern for building request bodies — reliable
- Auto-create folders via Graph API find-or-create pattern — eliminates manual setup
