Alfred went from v0.1.9 to v0.3.0 in 48 hours. Here's what changed — and why the project is no longer what it was.

**v0.1.9 was "AI agents that organize your Obsidian vault."**

**v0.3.0 is "the agent you can forget about."**

Same codebase. Completely different ambition. Alfred is now a personal agentic butler — infrastructure that connects to your AI runtime, maintains a knowledge graph, executes durable workflows, and communicates on your behalf. You set it up once and forget about it. That's the point.

Here's what shipped:

**Temporal execution engine.** Alfred now has a kinetic layer — a Temporal-based workflow engine that orchestrates agent work as durable, scheduled operations. Daily briefings. Inbox processing. Vault health sweeps. Write workflows in Python; the agent handles reasoning, Python handles control flow. If the worker crashes mid-workflow, it picks up exactly where it left off. `pip install alfred-vault[temporal]` to add it.

**Two live dashboards.** A Textual TUI (Python, integrated into `alfred up --live`) and an Ink TUI (React for terminals, standalone via `alfred tui`). Both show real-time worker status, pipeline progress, mutation feeds, and health indicators. The Ink version has sparklines, inline progress bars, and hex color theming.

**13 wiki pages of documentation.** Architecture, per-tool guides, vault schema reference, agent backend setup, configuration reference, CLI commands. All in `docs/` and pushed to the GitHub wiki.

**Production hardening.** Between v0.1.9 and v0.2.0: crash protection on all daemon loops, staggered worker startup, per-tool OpenClaw agent isolation, manifest retry loops, Ollama embedding resilience, and fixes across all four worker pipelines. Six patch releases of finding and fixing every way the system could die at 3am.

**Six-layer architecture.** Infra → Data → Semantic → Kinetic → Agent → Interface. The Obsidian vault is the semantic layer — a knowledge graph readable by both humans and agents. The Temporal engine is the kinetic layer. The agent backends (Claude Code, Zo Computer, OpenClaw) are the agent layer. Your runtime's channels (Telegram, WhatsApp, Slack, iMessage) are the interface layer. Each layer is independent and pluggable.

The short version: Alfred used to be four background workers for your vault. Now it's the infrastructure layer that turns any agentic runtime into a butler you can forget about.
